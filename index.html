>_ mental model · 你在跟什么对话

同一个 PsyClaw，三种工作方式

区别只在一件事：谁来决定下一步。从你主导到它自主，是一条滑杆。多数时候直接 psyclaw 进对话说人话就行。

chat 你主导

边聊边推进

讨论 + 工具按需调用、逐条批准。适合探索、还不确定下一步——决定权在你，它给建议。

run 明确流程

一次跑通某类流程

说「跑通 analysis / literature / meta / qualitative」，一次执行完、产出落盘。明确知道要做什么时用它。

auto 它自主

Ralph 式自循环

读状态 → 选任务 → 执行 → 独立验收 → 记录，循环推进。强制检查与不可逆决策仍会 fail-closed 暂停等你。

>_ research lifecycle · 它怎么看待「做研究」

七个阶段，每阶段一道门

一个阶段「完成」的标志不是跑完了，而是过了那道 gate。PsyClaw 在每阶段结束告诉你：过了没、缺什么、下一步该干嘛。

01
澄清 clarify
研究问题 → 17 项研究准备清单，逐项 grill 式敲定。
gate · 问题明确、变量可测
02
设计 design
选设计与分析方法，每个决策给文献背书。
gate · 设计能答问题
03
预注册 preregister
锁定假设与分析计划，出 OSF / AsPredicted 双格式。
gate · 探索 / 确证分明
04
文献 literature
检索 → 筛选 → 矩阵 → 综述。公开 API 直检或机构库浏览器桥接。
gate · 引用可核实
05

数据质量 → 描述 → 主分析 → 诊断 → 稳健性。统计外移：委托 scipy / pingouin 跑。
gate · 效应量 + CI、诊断齐

>_ how to · 直接说人话

对话示例

不用记命令、不用记 --flag——像跟研究助理说话一样告诉它你要什么，命令和文件它自己处理。切换看几个真实场景：

psyclaw · 对话
我想研究公正世界信念对大学生心理健康的影响，帮我起个头。
>_
正式开工前我先跟你确认两点，免得后面返工——① 这是探索性研究，还是有明确假设的确证性研究？② 心理健康用哪个量表测（生活满意度 SWLS？抑郁 CES-D？）。定了我帮你建独立分析文件夹、出检索计划。
确证性，结局用 SWLS 和抑郁。
>_
✓ 已建 bjw_study/，检索计划写好了（中英布尔式 + 纳入/排除标准 → notes/search_plan.md）。公开学术库我可以直接检索；知网/万方要登录的，我能驱动你已登录的浏览器去检，登录环节你来。先跑哪个？
@data/clean/bjw.csv 这份数据能做什么分析？
>_
读到了——124 行，变量有组别、PBJW、GBJW、SWLS、抑郁，加年龄/性别。适合做 PBJW / GBJW 对 SWLS 和抑郁的回归，控制人口学变量。统计我不自己算，生成一个委托 pingouin 的脚本跑相关 + 分层回归，报效应量和 95% CI。要现在跑吗？
跑。
>_
→ outputs/report.md。PBJW → SWLS β = .34, 95% CI [.18, .50]；正态与方差诊断齐，换 Spearman 复验结论一致。提醒：这是横断数据，只能说关联，别写成因果。
帮我把结果导成 Word，APA 格式，顺便看看能不能投。
>_
✓ → outputs/report.apa7.docx：TNR 双倍行距、中文字体前置（不会豆腐块）、两张图真嵌进去、参考文献悬挂缩进。质检也跑了 → JARS 报告完整、效应量 + CI 齐、引用都可核实。一点提醒：样本量偏小，讨论里建议补一句功效说明再投。
我这个结果 p = .048，能说公正世界信念显著提升了幸福感吧？
>_
统计上 p < .05 算显著，但两处得改：① 这是横断相关数据，「提升」是因果说法，换成「与幸福感正相关」更准；② 光报 p 不够，要带效应量和置信区间。还有 .048 很贴边界，如实报精确 p 就好——「边缘显著」这类话术我不会帮你写。
PHQ-9 多少分算中度抑郁？
>_
PHQ-9 总分 0–27，常用切分：5–9 轻度、10–14 中度、15–19 中重度、20+ 重度——所以 10 分起算中度。要我把条目和中国样本常模也调出来吗？
顺便，做中介效应该用什么方法？
>_
首选 bootstrap 法（偏差校正，5000 次）算间接效应的置信区间，别只看 Sobel。要做的话我按你的自变量 / 中介 / 因变量生成分析脚本。

这些都能直接说：量表查询 · 方法建议 · 实验设计 · 预注册 · 投稿评审 · 文献检索——想不起来怎么开口，就一句「psyclaw start」，它带你走一遍。

>_ design philosophy · 它对自己的约束

克制，是这套工具的性格

这些不是功能，是刻意的架构选择——决定了 PsyClaw 该做什么、绝不做什么。

externalized

统计外移

核里不实现任何统计算法。成熟库经同行审查、可复现——生成委托 scipy / pingouin / statsmodels 的脚本，或走 MCP，把「算」交给最合适的工具。

stdlib core

纯 stdlib 核

核心零第三方硬依赖，处处能跑。统计、浏览器、文献等重能力全外挂成 MCP，缺失时优雅降级。

integrity

学术诚信不可妥协

效应量 + CI 必报、相关 ≠ 因果、区分探索 / 确证、不 p-hacking、不杜撰引用——产出与门禁都照规范把关。

ralph loop

Ralph 自循环

auto 模式与开发流程同构：磁盘是记忆、上下文可抛弃，每轮从盘上重推待办、只做一件、拿真实产物验收、记状态、再来。

memory

三层记忆

对话上下文 / 会话状态（notes/，跟文件夹走）/ 长期画像与教训卡。用得越久越懂你，靠相关性路由 + 衰减防上下文膨胀。

capable

能力自知

它知道自己有导出、APA7 图表、质检这些现成能力，不重造轮子；手搓 docx / 裸 matplotlib 会被当场纠偏。
